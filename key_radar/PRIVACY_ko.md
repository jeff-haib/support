# KeyRadar 개인정보 처리방침

**시행일:** 2026-05-30 (초안)
**버전:** 0.3 (사업자 정보 확정, 변호사 검토 전)

> ⚠️ 이 문서는 변호사 검토 전 초안입니다. GDPR·CCPA·한국 개인정보보호법 적용 시 전문가 검토가 필요합니다.

## 한 줄 요약
KeyRadar는 사용자의 **API 키, vault 내용, 프로젝트 파일, 작업 내용**을 운영자(HAIB) 또는 결제 처리자(Polar)로 전송·저장하지 않습니다. 단, 유료 라이선스 검증 및 무료 체험 관리에 필요한 최소한의 라이선스 관련 정보는 처리됩니다(자세한 내용 §4).

## 1. 본 방침의 범위
본 방침은 KeyRadar 데스크톱 앱이 사용자의 데이터를 어떻게 다루는지 설명합니다. 사용자가 사용하는 제3자 provider(OpenAI, Anthropic 등)의 개인정보 처리는 각 provider의 방침을 따릅니다 — KeyRadar는 그것을 통제하지 않습니다.

## 2. 운영자가 직접 수집하는 정보
**없습니다.** HAIB는 자체 서버를 운영해 다음을 수집·전송·저장하지 않습니다.
- 사용자 식별 정보(이름, 이메일, IP 주소 등) — 단 §4의 Polar 결제 흐름에서 Polar가 보유하는 정보는 예외
- API 키
- vault 내용
- 프로젝트 파일·소스 코드·프롬프트
- 사용량 통계
- 텔레메트리·분석 데이터
- 크래시 리포트(자동 전송 없음)
- 사용 패턴

## 3. 사용자 기기에 저장되는 정보
다음은 사용자의 로컬 기기에만 존재하며 외부로 전송되지 않습니다.

3.1 **암호화된 vault (`vault.enc`)** — 사용자가 등록한 API 키. ChaCha20-Poly1305 등 강한 암호화로 저장.

3.2 **참조 메타데이터 (`refs/refs.json`)** — 어떤 프로젝트에 어떤 키가 적용되는지의 매핑.

3.3 **감사 로그 (`audit.log`)** — 키 복호화·공급 이벤트 기록(누가, 언제, 어떤 키). 사용자만 볼 수 있음.

3.4 **사용량 스냅샷 (`usage_daily.json`)** — provider에서 받아온 사용량 데이터의 일별 캐시.

3.5 **안전 보관함 (`safe_archive/`)** — 원본 `.env` 파일의 복사본(자동 삭제 안 함).

3.6 **`.keyradar.md` 좌표 파일** — 사용자 프로젝트 폴더에 생성됨. **키 값을 포함하지 않으며**, vault 참조 ID와 "KeyRadar 관리" 메타데이터만 포함.

3.7 **무료 체험 기록** — 체험 시작일과 동일 기기 이전 설치 흔적 탐지를 위한 로컬 표식. 운영자 서버로 전송되지 않습니다. (재설치·OS 재설치로 우회 가능 — 완벽한 남용 방지는 아님을 명시.)

3.8 **라이선스 키와 활성화 상태 캐시** — 사용자가 입력한 라이선스 키와 마지막 검증 결과의 로컬 캐시. 키 텍스트 자체는 Polar에 활성화·검증 시 전송됩니다(§4).

## 4. 결제·라이선스·무료 체험에 관한 데이터 (Polar 처리)
유료 라이선스 활성화 및 검증을 위해 다음 데이터가 **Polar(merchant of record)에서 처리**됩니다. HAIB 자체 서버는 이 흐름에 관여하지 않습니다.

4.1 **결제 시 (Polar에서 처리)**
- 결제 정보(카드, 청구 주소 등) — Polar가 직접 수집·보유합니다. HAIB는 카드 번호 등 결제 수단 정보를 받지 않습니다.
- 구매자 이메일 — Polar가 보유. HAIB가 운영 목적(라이선스 발급 확인, 고객 지원)으로 Polar 대시보드에서 일부 정보(구매 이메일, 라이선스 상태 등)를 조회할 수 있으나, HAIB 자체 데이터베이스에 복제·저장하지 않습니다.
- 거래 ID, 영수증 — Polar 보유.

4.2 **라이선스 활성화·검증 시 (KeyRadar가 Polar API 호출)** — KeyRadar는 다음 데이터를 Polar의 라이선스 엔드포인트로 전송합니다.
- 라이선스 키 텍스트
- 활성화 라벨(예: "MacBook Pro" — 사용자가 직접 보는 식별용)
- 활성화 메타데이터: 기기 식별을 위한 해시 처리된 device hash 등. (IP 등은 Polar 측에서 자체적으로 인식할 수 있습니다.)

4.3 **Polar가 보유하는 라이선스 정보**
- 라이선스 키 상태(유효/만료/철회)
- 활성화 인스턴스 목록(label, meta, 생성일)
- 검증 횟수, 마지막 검증 시각
- 활성화 한도(최대 2대), 만료일 등

4.4 **사용자의 라이선스 관리**
- Polar Customer Portal에서 사용자가 직접 활성화 인스턴스를 보고 비활성화할 수 있습니다.
- 환불·구독 취소 등 결제 관련 요청도 Polar Portal을 통해 처리됩니다.

4.5 **무료 체험에 관한 데이터**
- 무료 체험 기록은 §3.7대로 **로컬에만** 저장되며 운영자 서버로 전송되지 않습니다.
- 동일 기기 재설치 흔적 탐지는 로컬 표식 기반이며, 완벽하지 않습니다.

4.6 **Polar의 개인정보 처리**
- Polar의 개인정보 처리는 Polar의 정책(https://polar.sh)을 따릅니다. KeyRadar는 Polar의 처리를 통제하지 않습니다.

4.7 **향후 변경 시 갱신 약속**
- 현재 KeyRadar는 자체 라이선스/체험 서버를 운영하지 않습니다. 향후 HAIB가 자체 라이선스 서버 또는 체험 검증 서버를 도입할 경우, 본 방침을 *해당 도입 이전에* 갱신하고 사용자에게 고지합니다.

## 5. 외부로 나가는 통신
KeyRadar가 외부와 통신하는 경우는 다음으로 한정됩니다 — 모두 운영자 서버를 경유하지 않습니다.

5.1 **사용자가 등록한 provider 측** — 키 유효성 검증, 사용량/잔액 조회를 위한 공식 엔드포인트 호출.

5.2 **Polar 측** — §4의 라이선스 활성화·검증을 위한 엔드포인트 호출.

이 외에는 외부 통신이 없습니다. 자동 업데이트, 사용 통계 전송, 원격 로그 등을 수행하지 않습니다.

## 6. Provider 측 개인정보 처리
KeyRadar에서 검증·조회 요청을 보낼 때, 해당 provider는 자신의 정책에 따라 요청·IP·키 사용 패턴을 기록할 수 있습니다. 이는 KeyRadar의 통제 밖이며, 각 provider의 개인정보 처리방침을 확인하십시오.

## 7. 데이터 삭제·이전
7.1 **삭제:** Settings의 "모든 키 소거(Erase All)" 기능으로 vault·refs·감사 로그·임시 공급물·로컬 라이선스 캐시·체험 기록을 삭제할 수 있습니다. 안전 보관함의 원본 `.env` 삭제는 별도로 확인합니다. Polar 측 데이터(결제 영수증, 라이선스 기록 등)는 Polar Customer Portal에서 직접 관리하십시오.

7.2 **이전(다른 기기로):** 익스포트 기능으로 암호화된 vault 복제본을 다른 기기로 옮길 수 있습니다. 평문 익스포트는 제공하지 않습니다.

7.3 **앱 제거:** 앱을 제거해도 로컬 데이터 파일이 디스크에 남을 수 있습니다. 완전 삭제는 §7.1의 Erase All을 먼저 실행하거나, 앱 데이터 디렉토리를 수동 삭제하십시오.

## 8. 어린이 개인정보
KeyRadar는 일반 개발자를 대상으로 하며, 13세 미만(또는 관할 법령상 아동)의 개인정보를 의도적으로 수집하지 않습니다.

## 9. 사용자의 권리 (GDPR/한국 개인정보보호법 등)
- **HAIB 보유 정보:** HAIB는 사용자 개인정보를 자체 서버에 보유하지 않으므로, HAIB 측에서 직접 열람·정정·삭제·이동·처리정지의 대상이 될 정보가 존재하지 않습니다.
- **Polar 보유 정보:** §4의 결제·라이선스 관련 정보는 Polar Customer Portal을 통해 사용자가 직접 관리할 수 있습니다. 권리 행사는 Polar에 직접 청구해야 할 수 있습니다.
- **provider 보유 정보:** 각 provider에 직접 권리를 행사하십시오.
- **로컬 데이터:** 사용자가 직접 관리합니다.

## 10. 보안 사고 통지
HAIB가 사용자 정보를 직접 보유하지 않으므로 HAIB 측 데이터 유출 위험은 구조적으로 제한적입니다. 다만 KeyRadar 자체의 보안 취약점이 발견될 경우, 프로젝트 저장소를 통해 공지하고 패치를 배포합니다. Polar 측 사고는 Polar의 통지 정책을 따릅니다.
