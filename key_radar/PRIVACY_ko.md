# KeyRadar 개인정보 처리방침

**시행일:** 2026-06-02 (초안)
**버전:** 0.5 (Mac App Store 결제, 로컬 라이선스 메타데이터, vault 정체성 명확화, 변호사 검토 전)

> v0.5 (2026-06-03): 별도 vault password 제거. vault 정체성 명확화(보안 금고 아닌 로컬 정리·관측 저장소). 자동 검증 범위 "앱 실행 중에만" 명시. 정책 v0.9.4와 정합.

## 한 줄 요약
KeyRadar는 사용자의 **API 키, vault 내용, 프로젝트 파일, 작업 내용**을 운영자(HAIB) 또는 결제 인프라로 전송·저장하지 않습니다. Mac App Store 결제는 Apple이 처리하며, KeyRadar는 앱 잠금 해제에 필요한 로컬 라이선스 메타데이터만 저장합니다.

## 1. 본 방침의 범위
본 방침은 KeyRadar 데스크톱 앱이 사용자의 데이터를 어떻게 다루는지 설명합니다. 사용자가 사용하는 제3자 provider(OpenAI, Anthropic 등)의 개인정보 처리는 각 provider의 방침을 따릅니다 — KeyRadar는 그것을 통제하지 않습니다.

## 2. 운영자가 직접 수집하는 정보
**없습니다.** HAIB는 자체 서버를 운영해 다음을 수집·전송·저장하지 않습니다.
- 사용자 식별 정보(이름, 이메일, IP 주소 등)
- API 키
- vault 내용
- 프로젝트 파일·소스 코드·프롬프트
- 사용량 통계
- 텔레메트리·분석 데이터
- 크래시 리포트(자동 전송 없음)
- 사용 패턴

## 3. 사용자 기기에 저장되는 정보
다음은 사용자의 로컬 기기에만 존재하며 외부로 전송되지 않습니다.

3.1 **암호화된 vault (`vault.enc`)** — 사용자가 등록한 API 키. 로컬 정리·관측을 위한 저장소입니다. vault unlock secret은 macOS Keychain에 저장되거나(서명 빌드), dev/unsigned 빌드에서는 평문 파일로 저장됩니다(정직 고지: 임시 폴백). 어느 경우든 외부 서버로 전송되지 않습니다.

3.2 **참조 메타데이터 (`refs/refs.json`)** — 어떤 프로젝트에 어떤 키가 적용되는지의 매핑.

3.3 **감사 로그 (`audit.log`)** — 키 복호화·공급 이벤트 기록. 사용자만 볼 수 있음.

3.4 **사용량 스냅샷 (`usage_daily.json`)** — provider에서 받아온 사용량 데이터의 일별 캐시.

3.5 **안전 보관함 (`safe_archive/`)** — 원본 `.env` 파일의 복사본. 자동 삭제하지 않음.

3.6 **`.keyradar.md` 좌표 파일** — 사용자 프로젝트 폴더에 생성됨. 키 값을 포함하지 않습니다.

3.7 **트라이얼 및 라이선스 메타데이터 (`license_meta.json`)** — 라이선스 상태(`trial` / `expired` / `pro`), 트라이얼 시작 시각, 구매 정보. 이 데이터는 HMAC 서명으로 위변조 검출 가능하게 로컬 저장되며 사용자 기기 외부로 전송되지 않습니다.

3.8 **사용자 패스워드 없음** — 사용자가 별도의 vault password를 설정하지 않으므로, KeyRadar는 vault 패스워드를 저장·전송·복원할 일이 없습니다. 다른 기기로 옮기기 위한 export bundle password는 사용자가 직접 정하는 일회용 암호이며 KeyRadar에 저장되지 않습니다.

## 4. 결제·라이선스·트라이얼에 관한 데이터
KeyRadar는 Mac App Store 구매를 Apple In-App Purchase로 처리합니다.

4.1 **결제 시** — Apple App Store 결제 인프라가 결제 정보, 영수증, 거래 정보를 Apple 정책에 따라 처리합니다. HAIB는 카드 번호 등 결제 수단 정보를 받지 않습니다.

4.2 **라이선스 검증** — KeyRadar는 Apple StoreKit 검증 결과를 수신하고, 그 결과로 얻은 로컬 entitlement 상태를 `license_meta.json`에 저장합니다.

4.3 **무료 체험 데이터** — 무료 체험 기록은 §3.7대로 로컬에만 저장됩니다. KeyRadar는 트라이얼 강제를 위해 기기나 계정을 추적하지 않습니다.

4.4 **향후 직배포 채널** — App Store 외 직배포 채널이 도입될 경우, 결제 처리자와 데이터 흐름을 출시 전에 문서화합니다.

## 5. 외부로 나가는 통신
KeyRadar가 외부와 통신하는 경우는 다음으로 한정됩니다 — 모두 운영자 서버를 경유하지 않습니다.

5.1 **사용자가 등록한 provider 측** — 키 유효성 검증, 사용량/잔액 조회를 위한 공식 엔드포인트 호출.

5.2 **Apple App Store 인프라** — 구매, 복원, StoreKit 검증 흐름.

5.3 **자동 검증 범위** — provider 키 검증과 usage monitoring은 앱 실행 중에만 동작합니다. 앱을 닫으면 자동 검증도 중단됩니다.

이 외에는 외부 통신이 없습니다. 사용 통계 전송, 원격 로그 등을 수행하지 않습니다.

## 6. Provider 측 개인정보 처리
KeyRadar에서 검증·조회 요청을 보낼 때, 해당 provider는 자신의 정책에 따라 요청·IP·키 사용 패턴을 기록할 수 있습니다. 이는 KeyRadar의 통제 밖이며, 각 provider의 개인정보 처리방침을 확인하십시오.

## 7. 데이터 삭제·이전
7.1 **삭제:** Settings의 "모든 키 소거(Erase All)" 기능으로 vault·refs·감사 로그·임시 공급물·로컬 라이선스 메타데이터·트라이얼 기록을 삭제할 수 있습니다. 안전 보관함의 원본 `.env` 삭제는 별도로 확인합니다. Apple 측 구매 기록은 Apple을 통해 관리됩니다.

7.2 **이전:** 익스포트 기능으로 암호화된 vault 복제본을 다른 기기로 옮길 수 있습니다. 평문 익스포트는 제공하지 않습니다.

7.3 **앱 제거:** 앱을 제거해도 로컬 데이터 파일이 디스크에 남을 수 있습니다. 완전 삭제는 §7.1의 Erase All을 먼저 실행하거나, 앱 데이터 디렉토리를 수동 삭제하십시오.

## 8. 어린이 개인정보
KeyRadar는 일반 개발자를 대상으로 하며, 13세 미만(또는 관할 법령상 아동)의 개인정보를 의도적으로 수집하지 않습니다.

## 9. 사용자의 권리 (GDPR/한국 개인정보보호법 등)
- **HAIB 보유 정보:** HAIB는 사용자 개인정보를 자체 서버에 보유하지 않습니다.
- **Apple 보유 정보:** 구매 및 거래 정보는 Apple을 통해 관리됩니다.
- **provider 보유 정보:** 각 provider에 직접 권리를 행사하십시오.
- **로컬 데이터:** 사용자가 직접 관리합니다.

## 10. 보안 사고 통지
HAIB가 사용자 정보를 직접 보유하지 않으므로 HAIB 측 데이터 유출 위험은 구조적으로 제한적입니다. 다만 KeyRadar 자체의 보안 취약점이 발견될 경우, 프로젝트 저장소를 통해 공지하고 패치를 배포합니다.

## 11. 본 방침의 변경
본 방침은 변경될 수 있으며, 변경 시 앱 또는 저장소를 통해 공지합니다. 특히 텔레메트리, 크래시 리포트, HAIB 자체 서버, 새 결제 채널 도입 전에는 본 방침을 갱신합니다.
