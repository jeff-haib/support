# KeyRadar 트라이얼 정책 v0.5

> v0.5 (2026-06-03): 별도 vault password 제거. vault 정체성 명확화(보안 금고 아닌 로컬 정리·관측 저장소). 자동 검증 범위 "앱 실행 중에만" 명시. 정책 v0.9.4와 정합.

본 문서는 [이용약관](./TERMS_ko.md)의 보충 설명입니다. 충돌 시 약관이 우선합니다.

## 1. 트라이얼 기간

앱 첫 실행 시 자동으로 *3일* 무료 트라이얼이 시작됩니다. 사용자가 별도로 트라이얼 시작을 선택할 필요는 없습니다.

## 2. 트라이얼 중 제공 범위

트라이얼 기간 동안 모든 기능을 라이선스 보유자(Pro)와 동일하게 사용할 수 있습니다.

## 3. 트라이얼 만료 후

3일이 경과하고 라이선스를 구매하지 않은 경우:
- 앱은 만료(Expired) 상태로 전이됩니다.
- vault 접근이 차단되며, 모든 메뉴는 dimmed overlay로 가려집니다.
- 등록된 키 데이터는 디스크에 그대로 보존됩니다. 라이선스 구매 후 즉시 접근이 복구됩니다.
- 결제 버튼(Topbar 우상단)만 활성 상태로 유지됩니다.

## 4. 트라이얼 우회 방어

KeyRadar는 트라이얼 시작 시각을 로컬 파일(`license_meta.json`)에 HMAC 서명하여 저장합니다. 이 파일은 위변조 검출이 가능하나, 사용자가 vault 디렉토리를 완전히 삭제하고 앱을 재설치하면 트라이얼이 다시 시작될 수 있습니다.

이는 의도된 정책입니다:
- 기기 식별자 추적 없음 (개인정보 최소화 정책 정합)
- iCloud 등 추가 권한 요구 없음
- App Sandbox 환경 내에서만 동작

3일이라는 짧은 트라이얼 기간 자체가 재설치 우회의 비합리적 마찰 비용을 형성합니다.

## 5. 트라이얼 알림

다음 시점에 사용자에게 알림이 표시됩니다:
- 첫 실행 시: "3일 무료 트라이얼이 시작되었습니다" 모달
- 만료 1일 전: 토스트 알림
- 만료 시점: 결제 모달 자동 노출
